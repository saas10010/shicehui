# 批次详情页：学生 Tab 不展示“需处理”

## 背景
用户反馈：学生 Tab 出现“需处理”属于异常流程概念，且用 mock 写死不符合逻辑。

## 调整口径（原型）
- 学生 Tab 仅展示“初稿状态”：`处理中 / 可确认`（用于表达“初稿是否生成可看”）。
- “异常是否处理完/是否可快速确认”由异常 Tab 的“最终归属是否已保存”推导：
  - 若批次仍有未完成最终归属的异常（未保存），则禁用学生的“快速确认”，并在 tooltip 提示先去异常 Tab 归属并保存。
- 学生卡片的“已归属异常”按最终归属（保存态）统计，不再使用 `BatchStudentItem.exceptions` 的手填值。

## 代码位置
- `components/batches/batch-detail-tabs.tsx`：引入 `selected`（编辑态）与 `saved`（保存态），学生异常统计与快速确认基于 `saved` 计算。
